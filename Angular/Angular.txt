- signals
    -wrapper around a value that notifies the costumers when the value gets changed
    - writable or read only (computed signal)

    writable
        - signals are getter function meaning calling them reads their value.
        - writable signals have the type WritableSignal
        - create writable signal: const count = signal(0)
        - signals are getter function : count()
        - change value of writable signal: count.set(3)
        - operation to compute a new value from the previous one: count.update(value => value + 1);

    Computed 
        - const count: WritableSignal<Number> = Signal(0);
        - const doubleCount: Signal<Number> = computed(() => count() * 2);
        - are lazy evaluated and memoized meaning values are not computed until its read for first time and computed value is stored in cache.
        - cannot directly assign values to computed signal.( produces compilation error)
        -  TODO computed signal dependencies are dynamic.
        -  TODO - OnPuch
        - effect function ( slot in QT) - always run atleast once when one or more signal value changes.Always run asynchronously,during the change detection process.
            - used for logging data when it changes(debugging tool)
            - not to use - avoid using effects for propagation of state changes.
            - can only create effect() within an injection context(call effect within a component, directive, or service constructor)
            - TODO injector
            - effect is automatically destroyed when its enclosing context is destroyed.
            - effects return an EffectRef which can be used to destroy manually by calling .destroy() method.

        - by default signals use referential equality

HTTP client angular
    -Every front end applicationneeds to commmunicate with the cakend microservices to share the data over the HTTP protocol. In angular, this communication is done via the HTTP client.
    -Httpclient service is available iin the @angular/common/http package in the angaular framework.
    -for the Httpclient service to work,needs to import HttpclientModule in the application using the app module.
    -Different types of Httpclient methods provided 
    get(),post() - to post new client data to server,put() - to update data to server for the given resource or create one if it does nto exist.
    delete() - to delete item form the server for the given resource.
    patch() - to update part of the information for the given resource.

    -we will use a json server to understand http client and to install json server on our project we use.
    -npm install -g json-server and after this create a db.json file with some data.
    -start the server - json-server --watch db.json
