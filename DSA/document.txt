1.Loads the OpenAPI spec from com.hp.cdm.service.ioConfig.version.2.openapi.json and auto-generates up to 50 test cases (max_examples=50) for all GET endpoints under /cdm/ioConfig/v2/.
2.Authenticates each request using an OAuth2 bearer token obtained from the token fixture.
3.Calls and validates each generated request against the target device/simulator using case.call_and_validate(), which checks:
    -Response status codes match the spec
    -Response schemas match the spec
    -Content-type headers are correct
    -All checks from schemathesis.checks.ALL_CHECKS
4.Runs pre-call hooks via the setup_callbacks fixture, which invokes idle_state (currently a no-op) before each API call.
5.Uses deterministic generation — @seed(1) and derandomize=True ensure reproducible test cases across runs.
